# AGENTS.md

本文件规定 Codex 在 Shore Trace 项目中的长期工程规则。所有结论以当前代码、README、docs/HANDOFF.md、测试和实际运行为事实来源。

## 1. 项目目标

Shore Trace 是一个手机优先、本地优先的公考学习记录 MVP。

核心闭环：

```text
快速记录
-> 自动汇总
-> 发现薄弱点
-> 明确下一步行动
-> 第二天继续执行
```

第一版优先保证：

- 60 秒内完成记录。
- 10 秒内看出重点。
- 连续使用 7 天不过度增加填写负担。

## 2. MVP 边界

未经用户明确批准，不得擅自增加：

- 登录注册。
- 后端服务。
- 云同步。
- 多设备同步。
- IndexedDB。
- AI 复盘。
- 图表库。
- 会员系统。
- 复杂题型管理。
- 自定义错因复杂排行。
- 与当前任务无关的新功能。

## 3. 技术约束

当前项目技术栈和约束：

- Vue 3。
- Vite。
- Tailwind CSS，通过 `@tailwindcss/vite` 接入。
- Vue Router。
- Composition API。
- JavaScript。
- `localStorage`。
- 当前不使用 Pinia。

数据访问约束：

- 页面不得直接访问 `localStorage`。
- 数据读写必须经过 `src/repositories/studyRecordRepository.js`。
- 不得引入与实际代码不一致的技术栈说明或依赖。

## 4. 架构职责

- `src/views` 负责页面组织、路由参数处理和页面级交互编排。
- `src/components` 负责可复用 UI，不承载跨页面业务规则。
- `src/composables` 负责页面业务状态、保存状态和交互流程。
- `src/domain` 负责纯业务规则、校验、归一化和统计计算。
- `src/repositories` 负责数据读写、导出和本地存储异常保护。
- `src/utils` 负责日期等通用工具。

实现要求：

- 不在 Vue 页面重复实现已有业务规则。
- `domain` 层尽量返回结构化数据，不拼接 UI 文案。
- 不把存储、统计和展示全部写进同一个组件。
- 复用现有模块边界，避免为单次需求新增抽象。

## 5. 数据安全规则

- 不静默覆盖异常本地数据。
- 保存失败不能显示已保存。
- 未知或异常数据必须安全降级。
- 空模块不保存。
- 空记录不进入历史。
- 日期主键使用设备本地 `YYYY-MM-DD`。
- 不使用 UTC 日期生成业务日期。
- `createdAt` 首次保存生成，后续修改保留。
- `updatedAt` 只在成功保存后更新。
- 读取未知模块或未知字段时，优先保护可恢复数据，避免旧版本误删未来版本数据。

## 6. 修改范围规则

每次任务必须：

- 只修改与任务直接相关的文件。
- 不重构无关代码。
- 不顺带扩大功能范围。
- 不擅自增加依赖。
- 不把计划中的功能当成已完成。
- 以实际代码为事实来源。
- 保持改动小、清晰、可回滚。

## 7. 测试和验证

每次业务修改后至少执行：

```bash
npm test
npm run build
```

如果涉及页面交互，还需要说明哪些流程需要手动验证。

不得声称未实际执行的测试已经通过。若测试或构建无法执行，必须说明原因和剩余风险。

## 8. 对抗式审查

复杂任务完成后，需要对本次修改做范围受限的对抗式审查，重点检查：

- 功能遗漏。
- 边界条件。
- 数据丢失风险。
- MVP 范围漂移。
- 重复逻辑。
- 移动端交互。
- 测试覆盖不足。

审查只覆盖本次修改和直接依赖，不重新全面审计整个项目。发现真实问题后先修复，再重新执行相关验证。

## 9. 文档规则

- README 记录项目公开能力和使用方式。
- `docs/DECISIONS.md` 记录长期决策。
- `docs/HANDOFF.md` 记录当前进度和下一步任务。
- 不在多个文档重复同一份完整内容。
- `docs/HANDOFF.md` 应保持精简。
- 文档不得声称尚未实现的能力已经完成。
